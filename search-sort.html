Search and Sort.

What is search and sort?

In computer science, sorting and searching are fundamental operations. They are used to arrange data in a particular order or search for specific data in a collection of data. We will be going through some of the most common sorting algorithms as well as a personal favourite at the end! Selection Sort

Selection sort is a simple sorting algorithm that works by repeatedly finding the minimum element from the unsorted array and moving it to the beginning of the array on each iteration.

Let’s take a look at the code: 

public static void selectionSort(int[] arr) {
     int size = arr.length;
     for (int i = 0; i < size - 1; i++) {
          int minIndex = i;
          for (int j = i + 1; j < size; j++) {
               if (arr[j] < arr[minIndex]) {
                    minIndex = j;
               }
          }
          int temp = arr[minIndex];
          arr[minIndex] = arr[i];
          arr[i] = temp;
     }
}

Now let’s break down this code:
1. ‘size = arr.length’ gets the length of the parameter array ‘arr’.
2. The outer loops size-1 times.
3. ‘minIndex = i' initializes 'minIndex' to the index of the first unsorted element.
4. The inner loop iterates from 'j = i + 1' to 'j = size - 1'.
5. 'if (arr[j] < arr[minIndex])' checks if the current element at index 'j' is less than the current minimum element at index 'minIndex'. If it is, then 'minIndex = j' updates the index of the minimum element.
6. After the inner loop has finished, the minimum element of the unsorted subarray is swapped with the first unsorted element in the sorted subarray. This is done by swapping 'arr[minIndex]' with 'arr[i]'.
7. The outer loop continues until the entire array is sorted.


Insertion Sort

Insertion sort is another simple sorting algorithm that works by repeatedly inserting an element from the unsorted part of the array into its correct position in the sorted part of the array.

Let’s take a look at the code: 

public static void insertionSort(int[] arr) {
     int size = arr.length;
     for (int i = 1; i < size; i++) {
          int key = arr[i];
          int j = i - 1;
          while (j >= 0 && arr[j] > key) {
               arr[j + 1] = arr[j];
               j--;
          }
          arr[j + 1] = key;
    }
}

Now let’s break down this code:
1. ‘size = arr.length' gets the length of the input array 'arr'.
2. The outer loop iterates from ‘i = 1’ until i is equal to the size.
3. The while loop then compares each element to the value left of it until it finds one smaller.
4. Once the while loop finds an element that is smaller than the value to its left, it swaps the two values and continues to compare the newly swapped element with the value to its left until it finds a value that is not smaller.
5. The outer loop then moves to the next element and repeats the process, iterating through the entire array until all elements are sorted in ascending order.



Gnome Sort

Gnome Sort, also known as Stupid Sort, is a simple sorting algorithm that sorts an array by repeatedly swapping adjacent elements that are not in the correct order. This process is similar to the way a gnome sorts his flower pots in a garden, hence the name Gnome Sort.

Let’s take a look at the code: 

public static void gnomeSort(int[] arr) {
     int index = 0;
     int len = arr.length;
					
     while (index < len) {
          if (index == 0 || arr[index] >= arr[index - 1]) {
               index++;
          } else {
               int temp = arr[index];
               arr[index] = arr[index - 1];
               arr[index - 1] = temp;
               index--;
          }
     }
}
 Now let’s break down this code:

1. ‘int index = 0’ sets what is basically the starting position for the gnome (the first index of the array). The other integer variable ‘len’ is initialized to the length of the array.
2. The while loop iterates until ‘index’ is greater than or equal to ‘len’.
3. Inside the while loop, the program checks for if the current element is greater than or equal to the previous element. If so, the two values are sorted so the ‘index’ increments (the gnome takes a step forward).
4. If not, the program swaps the two values and ‘index’ is decremented.
5. The program (gnome) keeps taking a step backward and forward, checking if everything is sorted, until it reaches the end of the loop and everything is sorted!
